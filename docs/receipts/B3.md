## PLAN

Integrate `roboflow/supervision` (MIT licence) as the canonical bounding-box layer
for SPEC §3 VISUAL grounding method:

1. **Parse** any VLM/retrieval box output into a unified `sv.Detections` object via
   `parse_vlm_box()` — 5 supported formats: JSON dict (xyxy / xywh), JSON list,
   DeepSeek `<|box|>` token, plain CSV.
2. **Verify** via `verify_box_against_chunks()` — IoU >= psi=0.5 against Docling
   text/layout regions; if no chunk qualifies -> (False, "hallucinated") -> BLOCK.
3. **Annotate** verified boxes onto the page image with supervision BoxAnnotator +
   LabelAnnotator (same Detections object used for gate AND draw).
4. Declare `supervision>=0.20.0` in `requirements.txt`.

**Files affected:**
- `kernel/sidecar/ingest/bbox_verify.py`
- `kernel/tests/test_bbox_verify.py`
- `kernel/sidecar/requirements.txt`

**Gate command:** `pytest kernel/tests/test_bbox_verify.py -v`

---

## CRITIQUE

| Risk | Mitigation |
|------|-----------|
| VLM output format varies wildly | 5 regex/JSON fallbacks; CSV catch-all |
| Pixel-vs-normalised coord confusion | All coords clamped to [0,1] before IoU |
| Zero-area / inverted boxes leaking | x0>=x1 or y0>=y1 -> Detections.empty() -> BLOCK |
| supervision API breaks | Pinned >=0.20.0; uses stable Detections(xyxy=...) only |
| Missing supervision in requirements.txt | Fixed in this task |

## NOTES

- `annotate_page_image()` uses the **same** `sv.Detections` object for both
  the IoU gate and the BoxAnnotator draw call — satisfying the SPEC §3 requirement
  that "the same Detections object drives both the gate and the overlay draw".
- Zero-area / inverted boxes are blocked at parse time before IoU is computed.
- supervision 0.29.0.post0 is the installed version; >=0.20.0 is the floor.
