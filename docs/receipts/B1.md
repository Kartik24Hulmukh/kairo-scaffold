# B1 — DeepSeek-OCR2 native-grounding OCR adapter
Status: PASS
Date / commit: 2026-06-20

PLAN:
- Implement pluggable OCR backend interface with Docling (default for native PDFs), DeepSeek-OCR2 (3B VLM, for scanned pages), and OpenDataLoader (CPU fallback).
- In DeepSeek-OCR2 backend, parse grounding boxes from VLM tokens and normalize coordinates (scale 1000 -> [0, 1]).
- Modify `pdf_fastpath.py` (PyMuPDF subprocess) to output `is_raster = True` if page text char count < 5.
- Integrate per-page OCR dispatch in `app.py`: if `is_raster == True`, clear native chunks and run DeepSeek-OCR2 on the rendered page image.
- Verify IoU >= 0.5 for >= 90% of fields on a dynamically generated scanned PDF fixture in `test_ocr_backends.py`.

CRITIQUE:
- Isolated PyMuPDF strictly inside the `pdf_fastpath.py` subprocess to respect AGPL license boundaries.
- Designed backends with companion file loaders (`.grounding.txt` and `.docling.txt`) next to rendered page images for fast, deterministic offline testing.
- Handled coordinate normalizations carefully: DeepSeek token values (y0, x0, y1, x1) are parsed and correctly scaled to normalized [0, 1] page coordinates.
- Set a low threshold (5 characters) for raster classification to prevent native PDFs with brief text (e.g. `test.pdf`) from being incorrectly routed to OCR.
