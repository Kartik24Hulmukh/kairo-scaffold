## CRITIQUE

- **keyring OS package dependency offline build**: Bringing keyring-rs crate into the Rust CLI would require network access to download it, which fails in offline mode. Solved by delegating Rust subcommand calls directly to the virtualenv Python keyring integration, avoiding new Rust crate compilation.
- **Mocking dynamic secrets in benchmark**: Dynamic import of secrets inside the benchmark script requires patching the secrets module function `kernel.sidecar.models.secrets.get_api_key` at call-time.
- **Hugging Face offline connection errors**: During benchmark/test execution, `sentence-transformers` tries to reach huggingface.co to check for model configuration updates. Solved by setting `HF_HUB_OFFLINE=1` and `TRANSFORMERS_OFFLINE=1` to enforce pure local loading from disk cache.
