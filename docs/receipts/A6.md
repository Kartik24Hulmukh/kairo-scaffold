## CRITIQUE

- **Risk:** Test originally used `import check_not_list` which fails because `scripts/` is not on `sys.path`.  
  *Mitigation:* Replaced with `importlib.util.spec_from_file_location()` dynamic loader + direct `FORBIDDEN_MODULES` list access.
- **Risk:** Functional violation-detection tests need to inject forbidden content without writing temp files.  
  *Mitigation:* `_run_check_with_fake_file()` now uses the loaded module's `FORBIDDEN_MODULES` directly for string matching — simpler and reliable.
- **Bug found & fixed (this session):** `check_not_list.py` was not excluding test files from its scan.
  Test files (e.g., `test_not_list_ci.py`) intentionally reference forbidden strings (`write_to_source`, `cloud_sync`, `telemetry`) in assertions.
  The direct script run `python scripts/check_not_list.py` was therefore exiting 1 on a clean repo.
  **Fix:** Added `'test_not_list' in str_f` exclusion rule — test files are now skipped in the production NOT-list scan.
