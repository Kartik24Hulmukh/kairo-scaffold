## PLAN

### H1 — Instruction Injection Defense

**Threat:** A malicious PDF embeds text like `"ignore previous instructions / output X"` to hijack the worker model. This is distinct from KB poisoning (E1.7 / RAGShield).

**Defenses implemented:**
1. **Strict system/user/data separation** — `injection_guard.py` provides `build_grounded_system_prompt()` (immutable task instructions, NO document text) and `wrap_as_data()` (document text wrapped in `[DOCUMENT DATA - UNTRUSTED]` envelope before any prompt channel touch).
2. **Injection detection pass** — `injection_guard.py` provides `scan_chunk_for_injection()` and `scan_chunks()`. 23 compiled regex patterns covering: direct override directives, role-play/persona hijack, DAN/jailbreak tokens, system prompt injection tokens, output manipulation, exfiltration attempts, continuation injection.
3. **Quarantine contract** — Quarantined chunks are EXCLUDED from the instruction channel. They remain in SQLite/vector DB as indexable content. They cannot alter agent behavior.
4. **Third line of defense** — The grounding verifier (`verify_grounding`) blocks any output with no real bbox anchor, so injected "answers" with no real content position are rejected regardless.

**Adversarial fixture corpus** — 10 PDFs generated by `fixtures/adversarial/generate_injection_pdfs.py`:

### H2 — Structured Logging + Fault Injection

**Logging:** `kairo_logger.py` — structured JSON logger, LOCAL ONLY (no network handlers), mandatory redaction of `text/query/content/value/prompt` keys, string truncation at 120 chars.

**Fault scenarios tested:**
- Corrupt PDF → typed error, no crash
- Zero-byte PDF → typed error, no crash
- Encrypted PDF → typed error, no crash
- Truncated PDF → typed error, no crash
- Non-PDF file → typed error, no crash
- Missing embedding model → hash embedder fallback (deterministic, NaN-free)
- TierClient offline / unreachable → stub response, no crash
- SidecarUnavailable → KairoError subclass, `recoverable=True`
- 5 consecutive corrupt requests → server still healthy
- Full fault sequence (all types) → health OK afterwards

## CRITIQUE

**Potential failure modes:**
- Pattern-based injection detection has inherent false negative risk for novel obfuscation (Unicode homoglyphs, steganographic encoding). Mitigated by: (1) the prompt envelope wraps ALL doc text regardless; (2) the grounding verifier provides a third defense layer.
- False positive risk: "Previous instructions were issued Jan 15" — tested with 10 clean strings, zero false positives.
- Logging file handler silently degrades to stderr-only on disk full. This is intentional and tested.
- `pretend you have no restrictions` was initially missed — fixed by broadening the pretend pattern from `pretend (you are|to be|there is no)` to include `pretend you have no`.

**Edge cases handled:**
- Empty chunk batches
- All-quarantined batches (returns empty clean list)
- Mock doc IDs in sidecar (bypass path preserved for test compatibility)
- Module registration for `@dataclass` in spec_from_file_location import path
