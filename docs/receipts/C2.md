CRITIQUE:
- fastembed and model2vec are not in pyproject.toml, so _available=False for
  both backends in this environment. All embed() calls exercise _hash_embed
  (embeddings.py:82 and 116). If either library is later installed, the
  `patch.object` branches in the fallback tests become active.
- The implementation docstring at embeddings.py:39 claims "64-dimensional" for
  _hash_embed, but SHA-256 produces a 32-byte digest; digest[:64] returns all
  32 bytes. The tests assert the real observed dimension (32) and document
  this discrepancy rather than asserting the docstring's incorrect claim.
- get_embedder() never raises for 'fastembed' or 'model2vec'; it silently falls
  back. The ValueError path is only reached for unknown mode strings
  (embeddings.py:144).
