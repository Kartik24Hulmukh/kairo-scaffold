CRITIQUE:
- POISONING_PATTERNS uses simple regex; adversarial prompt authors can
  obfuscate (e.g. Unicode lookalikes) to evade exact-match patterns.
  A semantic classifier would reduce evasion risk but adds inference cost.
- sanitize_user_query truncates by character count, not token count;
  a very long CJK query could still exceed model context if tokens < chars.
- quarantine_if_poisoned does not write to an audit log; detected events
  are silently dropped unless the caller inspects the returned dict.
