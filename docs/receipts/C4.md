# C4 -- Eval Harness (Ragas/DeepEval-style grounding metrics)
Status: PASS
Date / commit: 2026-06-19

PLAN:
- Implement EvalHarness class in bench/eval_harness.py with run(qa_pairs) -> EvalReport
- Three metrics: grounded_answer_rate, citation_hallucination_rate, refusal_correctness
- Metrics are deterministic (text-overlap / n-gram based), no live model required
- EvalReport is a dataclass with as_dict() for JSONL serialisation
- Results appended to bench/history.jsonl in append mode on every run()
- CLI entry-point invoked by make eval (Makefile line 70)
- 12 pytest tests in kernel/tests/test_eval_harness.py covering helpers + integration

CRITIQUE:
- Grounded-answer detection uses 5-gram overlap; answers shorter than 5 words fall back
  to keyword check, which is weaker but avoids false negatives on short factual answers.
- questions.json schema differs from eval harness qa_pair schema; CLI adapts on load.
- Refusal correctness only covers unanswerable questions (expected_answer == empty);
  answerable pairs do not affect this metric, correct by design.
- history.jsonl is not schema-validated on read; reader should check report key.
