## PLAN

Implement the SPEC §4 CRITICAL pattern:

Phase 1 — Free-form reasoning scratchpad (UNCONSTRAINED):
  The model writes a chain-of-thought section. No token constraints applied.
  Scratchpad is separated from Phase 2 by a fixed delimiter.

Phase 2 — Constrained final JSON object:
  The decoder emits a JSON object that MUST conform to the Pack JSON Schema.
  Token-healing strips LLM preamble/postamble before parsing.

Pluggable backends:
  * outlines (default) — ~5x faster, zero overhead per AWS
  * xgrammar — selectable via KAIRO_CONSTRAINED_BACKEND=xgrammar
  * llguidance — selectable via KAIRO_CONSTRAINED_BACKEND=llguidance

Files:
  kernel/sidecar/models/constrained_decoding.py — full implementation
  kernel/tests/test_constrained_decoding.py — 47-test suite
  kernel/sidecar/app.py — wired validate_and_heal into /extract and /ask

Gate command: pytest kernel/tests/test_constrained_decoding.py -v

---

## CRITIQUE

| Risk | Mitigation |
|------|-----------|
| Over-constraining scratchpad | Phase 1 is NEVER constrained; only Phase 2 is |
| Required fields dropped | _gen_val() guarantees required fields always; + post-validate |
| Old fuzzer bug (required skipped) | Fixed: required fields emitted before optional random selection |
| enum/minimum violations | _gen_val() samples from enum; clamps minimum/maximum |
| Token-healing edge cases | Brace-counting parser handles nested objects, escaped quotes, BOM |
| Missing Outlines/XGrammar/llguidance | Backends degrade gracefully to schema-guided fuzzer in CI |
| app.py char_span format change | validate_and_heal rebuilds anchor dict with char_start/char_end |

  Pydantic validation — it ensures enum values and confidence ranges are
  correct even if upstream logic produces unexpected values.
- The scratchpad delimiter ("===BEGIN_STRUCTURED_OUTPUT===") allows
  consumers to extract Phase 1 reasoning independently of Phase 2 JSON.
