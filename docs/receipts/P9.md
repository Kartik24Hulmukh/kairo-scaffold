## PLAN

### P9-A: Cross-Platform Overlay Abstraction
- Create a unified `OverlayPlatform` trait in `platform/mod.rs` to abstract window transparency, click-through configuration, pinning, and CORS headers.
- Implement platform-specific handlers for Windows (WebView2), macOS (WKWebView), and Linux (WebKitGTK).
- Refactor `main.rs` to initialize the overlay using the platform trait via `platform::setup_overlay`.
- Update `Cargo.toml` with target-specific dependencies (e.g. `windows` for Windows, `objc2`/`objc2-app-kit` for macOS).
- Configure window options in `tauri.conf.json` with transparency, decorations off, and always-on-top flags.
- Register `kairo-img://` custom URI scheme utilizing a direct context/request closure for Tauri v2 compatibility.

### P9-B: Versioned Migration System
- Design and implement a versioned SQLite database migration system (`user_version` PRAGMA) mapping schemas v1 to v2.
- Implement Python migration manager (`kernel/sidecar/retrieval/migration.py`) and Rust migration manager (`overlay/src-tauri/src/migration.rs`).
- Wire the Python migration runner to execute automatically on sidecar startup before any database operations.
- Create automated test suite (`kernel/tests/test_migration.py`) to verify version increments, idempotency, and error handling.

## CRITIQUE

- **Tauri v2 dependencies compilation error**: Tauri v2 Cargo build failed initially due to a non-existent `"transparent"` feature flag on the `tauri` package. Solved by removing it since transparency is configured via `"transparent": true` in `tauri.conf.json` and handled natively by Wry.
- **AppKit feature error**: The macOS build dependency `objc2-app-kit` had a non-existent feature `"NSWindowLevel"` defined in `Cargo.toml`. Solved by removing `"NSWindowLevel"` from features list since it's already included in the `"NSWindow"` module.
- **Tauri sidecar binary error**: Tauri's build script requires any configured sidecar binary (in `tauri.conf.json`'s `externalBin`) to physically exist under `src-tauri/sidecar/kairo-sidecar-<triple>.exe`. Resolved by writing a mock sidecar file for compiling check validation.
- **Tauri v2 URI protocol change**: Tauri v2 `register_uri_scheme_protocol` takes a closure that accepts `UriSchemeContext` and `Request`. The signature was mismatched. Solved by registering a closure directly in `main.rs` that borrows `request` and forwards it to `asset_protocol::serve_asset`.
