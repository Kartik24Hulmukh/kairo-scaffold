# G4 — Schema/index migration across updates
Status: PASS
Date / commit: 2026-06-19

PLAN:
- Implement a MigrationManager module in the sidecar retrieval system.
- Check schema versions in a 'schema_version' table and run numbered SQL migrations.
- Support a v1->v2 migration that adds 'page_index' to the chunks table.
- Verify that it works idempotently, recovers from failures, and is backward-compatible.

CRITIQUE:
- A new database might not have the chunks table. We must skip the column-adding step if the table doesn't exist yet to avoid crashes.
- Version table updates are kept transactional per migration step.
- Empty directories are automatically created before initialization.
